# Contributing to VAR-Core

VAR-Core is an open standard. Contributions to the specification, reference
implementations, and adapters are welcome.

## Spec changes

Open an issue labelled `spec` before submitting any PR that changes normative
language (MUST/SHOULD/MAY) in `docs/spec/`. Normative changes require discussion
and a rationale comment before merging.

## Implementation contributions

PRs welcome for bug fixes, new adapters, test coverage improvements, and
ports to other languages (see `python-verifier/` for a reference port).

## New adapters

New framework adapters belong under `packages/adapter-<framework>/`.
Follow the structure of `adapter-openai` as the reference.

## Code of Conduct

Be precise. Be constructive. Cite evidence.
